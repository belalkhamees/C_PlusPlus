1. What are the four concepts of OOP with a short explanation for each? 


Q1:

1- encapsulation -> bundling related fields and methods inside the same class.

2- abstraction -> that class we cant create object from it , it use as base class for child classes to make common interface to them , class become abstracted when it have at least one pure virtual function .

3- inheritance -> when we have the same attributes and functions between classes we use inheritance to achieve the code reusability, we can do any modify in base class and it will be affect in child classes, each derived class have it own local attributes and methods.

4- polymorphism -> an oop concept it refers to the ability of variable or method or object to take a multiple form, it have to types , compile time like function overloading and runtime like function overriding.

5. What is the difference between overloading and overriding?          

Q5:

-Function overloading -> means same name different signature (signature is: number & type & order of arguments).

-Function overriding -> means same name same signature it happens in inheritance when we want to use same function to do different behavior in each derived class.
